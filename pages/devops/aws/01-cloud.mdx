## 1. 클라우드

![wikipedia-cloud](https://upload.wikimedia.org/wikipedia/commons/b/b5/Cloud_computing.svg)

- `클라우드` : 전산 자원들을 공유하는 기술과 도구의 집합
  - Cloud라고 부르는 영어 단어는 우리 말로 ‘구름’이라는 뜻
  - Cloud라는 단어는, Cloud Computing을 짧게 줄여서 부르는 말
- `클라우드 컴퓨팅`
  - 구름처럼 **하늘을 둥둥 떠다니며 전세계 어디서든지 컴퓨팅 자원을 사용할 수 있다**는 의미
  - Data 저장과 접근을 인터넷으로 언제 어디서든 접근 가능한 기술
  - 사용한 만큼만 과금 되는 전산기반 시설들
  - 전산 하드웨어 장비들의 가상화(virtualization) 기술
- `시설` : infrastructure라고 부름. 줄여서 infra라고 함
- `전산 자원` :
  - server, storage, network 같은 물리 하드웨어 뿐만 아니라,
  - 플랫폼과 소프트웨어까지 포함하는 개념

## 2. 클라우드 컴퓨팅의 탄생

- CPU는
  - 사람으로 치면 두뇌에 해당하는 것이고,
  - 코어가 많을 수록 연산 능력이 뛰어남
- 예전 CPU는 **싱글 코어**가 대부분이었다.
- 발전하면서 코어가 2개인 **듀얼코어**가 나옴
  - 인터넷의 등장과 함께 이렇게 컴퓨터들끼리 **데이터**를 주고 받을 수 있게 됨.
  - 처음에는 인터넷 속도가 조금 느렸지만, 그래도 데이터를 주고 받을 수는 있었음
- 인터넷 역시 발전하면서 속도가 빨라지고, 이러한 **데이터 공유 속도**에 점차 빨라짐
- 멀티코어를 가진 컴퓨터들이 늘어나면서, 컴퓨팅 자원이 많이 남음
- 남는 컴퓨팅 자원을 잠시 사용하지 않는 동안에만 빌려주고, 돈을 받는 수익모델이 등장
  - e.g. Airbnb : 안쓰는 집이나 방을 빌려주고 숙박비를 받는 모델

---

### 2.1 가상화

Virtualization(가상화)

- `가상화` : 서버, 스토리지, 네트워크 같은 하드웨어를 소프트웨어 형태로 추상화 시킨 것
- `가상머신` : 가상화 기술을 사용해서 하드웨어로 이뤄진 컴퓨팅 자원들을 소프트웨어로 구현한 것
  - **Virtual Machine**이라고 하고, 흔히 줄여서 **VM**이라고 부름
- 가상화 기술 덕분에 맥OS에서 윈도우를 실행하고, 윈도우에서 리눅스를 실행 가능
- VM을 인터넷을 통해서 원격으로도 제공 가능해짐
- `클라우드 서비스`
  - 가상화 기술을 이용해서 하드웨어를 소프트웨어로 만든 Virtual Machine을
  - 인터넷을 통해서 빌려주는 서비스

## 3. 어디까지 빌려주나

인터넷을 통해서 컴퓨팅 자원을 빌려줄 수 있게 되었는데, 어디까지 빌려줘야 하나?

![pizza as 서비스](https://media.licdn.com/dms/image/C4E12AQH3p44mG-q4Wg/article-inline_image-shrink_1000_1488/0/1520041067201?e=1710374400&v=beta&t=HP4YWW8O6FSiXhKW-s5OjRbOvoP-5OdKzqgYfVbLQn8)

- cf. https://www.linkedin.com/pulse/20140730172610-9679881-pizza-as-a-service
- `홈메이드 피자` : Cloud 안씀
- `냉동 피자` : Infrastructure(시설)만 대여
- `배달 피자` : Platform까지 대여
- `음식점 피자` : Software 통째로 대여

![IaaS-PaaS-SaaS](https://www.redhat.com/rhdc/managed-files/iaas-paas-saas-diagram5.1-1638x1046.png)

AWS는 이 중에서 세 가지(Infrastructure, Platform, Software)를 포함한다.
Infrastructure만 빌릴 수도 있고, 플랫폼을 통채로 빌릴 수도 있으며, 소프트웨어까지 완성제품으로 빌릴 수도 있다.
이렇게 어디까지 빌려주는 지에 따라서 `IaaS`, `PaaS`, `SaaS` 등 다양한 이름으로 불린다.

- `On-Premises(온프레미스)` : Cloud 안씀, 모든 걸 직접 구축한다.
- `IaaS(Infra)` : Infrastructure(시설)만 (대여, 가상화, 서버, 저장소, 네트워킹)
  - Infra as a service의 약자
- `PaaS(Platform)` : Platform까지 대여
- `SaaS(Software)` : Software 통째로 대여

## 4. 클라우드를 써야하는 이유

### 4.1 개인

- 개인이 집에서 직접 서버를 운영하려면, (On-Premise)
  - 발전기 설치, 높은 초기 구축 비용, 용량의 한계, 많은 노력이 필요
- 개인이 클라우드 컴퓨팅을 사용하게 되면, (Cloud Computing)
  - 필요할 떄 언제나, 사용한 만큼만 지불, 유연한 용량, 적은 노력이 든다.

---

### 4.2 기업

- 기업에서 직접 IDC를 운영하는 경우, (On-Premise)
  - 하드웨어 구매, 높은 초기 투자비용, 한정된 용량, 많은 소요시간이 필요
- 기업이 클라우드 컴퓨팅을 사용하게 되면, (Cloud Computing)
  - 필요할 떄 언제나, 사용한 만큼만 지불, 유연한 용량, 적은 노력이 든다.

## 5. 클라우드 컴퓨팅 장점

1. `초기 선투자 비용없음`
2. `운영비용 절감 `
   - e.g. 전력비용, 항온항습 비용, 임대료, 운영 및 관리 인건비, 라이선스 비용, 향후 증설비용
3. `탄력적인 운영 및 확장`
   - 자동으로 필요한 자원을 증설 및 감소 가능
   - 최적의 성능 및 안전성을 제공
4. `속도 및 민첩성`
   - 온프레미스 환경은 인프라를 준비하는데 몇주가 걸리고, 실패할 수 있고, 혁신 속도가 느림
   - 클라우드는 몇 분내 인프라를 준비하고, 실패비용이 낮고, 많은 혁신이 가능
   - 클라우드 기반 글로벌 서비스 e.g. Airbnb, Supercell, etc...
5. `비즈니스에만 집중 가능`
   - 하드웨어를 구매, 설치나 소프트웨어를 구매, 설치 등의 작업을 생략하고,
   - 그 시간에 자체 비즈니스에만 집중
   - e.g. 2014년에 Airbnb의 사용자가 대략 1500만명이었지만,
   - EC2 인스턴스. 즉 가상 서버 1300대를 사용했지만, 운영팀은 5명
6. `글로벌 확장`
   - 전 세계 어디든 수 분 내 확장해 서비스 구축 가능
