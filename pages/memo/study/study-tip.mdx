# 스터디 잘 운영하려면

## 1. 개발 스터디

- 스터디 목적 : 이력서/포트폴리오 10-20만, 참여를 잘 안함
- 소속감이 없고, 혼자 공부하는 외로움/고립
- 벌금제도는 생각보다 효과없다. 그냥 내고 말지 마인드
- 서로 친해져야 한다. (웃으면서 넘어가는 분위기)
- 모임에 자주 참여
- 같이 공부x, 오늘한 거 공유(좋았던점/ 안좋은점)
- 최대 6명, 4~5명정도

4명

1. 자기소개
2. 오늘 한거 공유/자기가 하는거 공유
3. 포트폴리오 아이디어 로직 -> 어떻게 구현할 건가?
4. 이력서 마무리 - 피드백
