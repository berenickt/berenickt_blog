# 1. 개발 블로그 종류

사람들마다 개인 블로그를 가지고 올리는 주제는 다양하지만,
개발을 메인 주제로 삼는 블로그는 크게 2종류로 나뉩니다.

- 기술 블로그
- TIL(Today I Learned)

---

# 2. 기술 블로그

- 특정한 Stack, Framework, method 등을 주제로 포스트를 작성한 블로그
- 다른 사람들에게 설명을 주목적으로 작성
- 자신만의 언어, 예시 코드로 정리하여,
  - 다른 사람들도 쉽게 이해할 수 있도록 작성하는 것이 좋은 기술 블로그
- e.g. [카카오 기술 블로그](https://tech.kakao.com/)
- e.g. [네이버 기술 블로그](https://d2.naver.com/home)
- e.g. [우아한형제들 기술 블로그](https://woowabros.github.io/)
- e.g.[Velopert](https://velopert.com/)
- e.g. [쿠팡 기술 블로그](https://medium.com/coupang-tech/technote/home)

---

# 3. TIL 블로그

- Today I Learned의 약자로 "오늘 내가 공부한 것"을 주제로 글을 작성하는 것
- 자신이 배운 것을 자신만의 언어로 정리
- 일기처럼 느낀점을 작성할 수도 있고, 복습한 내용을 정리해도 됨
- e.g. [초보몽키의 개발공부 블로그](https://wayhome25.github.io/til/2017/08/14/TIL-for-6-months/)
- e.g. [JBEE님의 블로그](https://jbee.io/essay/how_do_i_study_2/)
- e.g. [진유림님의 블로그](https://milooy.github.io/)
- e.g. [진유림님의 TIL](http://milooy.github.io/TIL/)

# 4. 블로그 플랫폼

## 4.1 velog

- 장점
  - 개발자들을 위한 전용 블로그
  - 마크다운 사용
- 단점
  - 팔로우 기능이 없어 타인과 소통하기 힘들다.
  - 댓글 알림 기능이 없다.
  - 디자인을 바꿀 수 없다.

---

## 4.2 Tistory

- 장점
  - 애드센스 삽입이 가능
  - 다양한 블로그 스킨과 디자인 변경 가능
  - 최신댓글, 방문자 수, 깃허브 잔디 등을 보여줄 수 있다.
- 단점
  - 코드 태그가 예쁘지 않다.
  - 마크다운 안됨

---

## 4.3 Notion

- 장점
  - 자유로운 디자인, 레이아웃
  - 마크다운 사용
  - 토글, 링크 삽입 등 다양한 기능을 쉽게 이용
  - 공동 작업 가능
- 단점
  - 구글 검색이 안 된다.
  - 다른 사람과 소통이 힘듬

---

## 4.4 Medium

- 장점
  - 세계적으로 큰 사이트이며, 해외 유저가 많다.
  - 구독 기능이 있다.
- 단점
  - 애드센스가 불가능
  - 부분 무료화이기 때문에 결제해야 읽을 수 있는 글이 있다.

---

## 4.5 github blog

- 장점
  - 다양한 스킨이 있으며, 스킨에 따라 기능도 모두 다르다.
  - 깃허브 잔디가 생긴다.
  - 마크다운 사용
- 단점
  - 초보자에게 어렵다.
  - 개인 공간인 느낌이 크기에 다른 사람과의 소통이 어렵다.
  - 구글 검색이 힘들다.

---
