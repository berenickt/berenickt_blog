## 1. Tailwindcss

https://tailwindcss.com/docs/installation

- Utility-first CSS framework,
  - 즉 미리 정의된 CSS의 특성들을 `class` 를 통해 불러와 적용시키는 형태의 CSS 프레임워크
  - 원하는 `html` 요소에 `class` 을 부여해주면 바로 사용이 가능하다
- `Tailwind CSS`: CSS 프레임워크. 미리 정의된 클래스를 사용하여 HTML 스타일을 쉽게 적용할 수 있음
  - e.g.) bg-blue-500 클래스를 적용하면, 따로 CSS 스타일링 없이도 파란 배경을 만들 수 있음
  - e.g.) px-4, mx-4 클래스들을 적용하면, 따로 CSS 스타일링 없이도 padding, margin 관리 가능
- `클래스 기반`: Utility-first라는 개념하에 클래스 이름으로 스타일링 가능. HTML과 CSS의 분리를 최소화하고 재사용성을 높임
- `커스텀 설정`: tailwind.config.js 파일을 통해 사용자 정의 스타일 및 구성 제공
- `일관된 디자인`: 미리 정의된 클래스로 일관서 있는 디자인 구현 가능
- `반응형 스타일링`: 미디어 크기에 맞는 반응형 디자인을 지원하며, 클래스 접두사로 편리하게 제어 가능
- `풍부한 자료`: 거대한 커뮤니티와 풍부한 자료, 그리고 Tailwind를 이용한 여러가지 컴포넌트들
- Tailwind CSS 공식 문서: https://tailwindcss.com/docs/installation

> cf. VSCode 확장기능
>
> - Tailwind CSS IntelliSense : 클래스명 자동완성
> - Tailwind Docs : 문서 찾아볼 떄 Ctrl + Shipt + P로 검색

### 1.1 Tailwind 주요 개념

- Utility-First Fundamentals (유틸리티-퍼스트 기본 원칙)
- Handling Hover, Focus, and Other States (호버, 포커스 및 다른 상태 처리)
- Responsive Design (반응형 디자인)
- Dark Mode (다크모드)
- Reusing Styles (스타일 재사용)
- Adding Custom Styles (커스텀 스타일 추가)
- Functions & Directives (함수 및 디렉티브)
- cf. Tailwind CSS 공식 문서 참고: https://tailwindcss.com/docs/utility-first

---

### 1.2 Tailwind CSS 장단점

[장점]

- 빠른 개발: 클래스 기반 스타일링을 통해 보다 빠르게 개발할 수 있음
- 커스터마이징: Tailwind 설정 파일을 통해 프로젝트에 맞게 스타일 조정 가능
- 재사용성: 클래스를 재사용해서 일관된 디자인 구현 가능
- 반응형: 클래스 접두사로 반응형 디자인을 간편하게 구현할 수 있음 (ex. lg:text-red-500, md:p-4)

[단점]

- 파일 크기: Tailwind를 사용해서 구현하면 클래스가 복잡해지고, 파일 크기가 커질 수 있음
- 학습 곡선: 처음에는 클래스 기반 스타일링에 익숙하지 않아, 초기 학습 비용이 많이 들어감

## 2. 기본 사용법

https://tailwindcss.com/docs/installation

```sh
npm install -D tailwindcss
npx tailwindcss init
```

tailwindcss를 사용하고자 하는 경로, 파일 작성(**확장자명은 띄워쓰면 안됨**)

```js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ['./src/**/*.{html,js,jsx,ts,tsx}'],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

전역 CSS에 다음 코드 추가 (CRA에서는 index.css 맨 위에 추가)

```css
/* tailwind 기본 스타일 및 확장된 스타일 주입 */
@tailwind base;

/* tailwind 기본 컴포넌트 클래스 및 확장된 컴포넌트 클래스 주입 */
@tailwind components;

/* tailwind 기본 유틸리티 클래스 및 확장된 유틸리티 클래스 주입 */
@tailwind utilities;
```

## 3. Next.js에서 Tailwind CSS 사용

(1) yarn으로 Tailwind CSS 설치 후, Tailwind CSS 구성 파일 생성을 위해 tailwindcss init 커맨드 입력

```bash
yarn add -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

(2) PostCSS 설정 파일에 Tailwind 추가

```js
// postcss.config.js
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}
```

(3) TailwindCSS를 적용할 템플릿 파일 경로를 `tailwind.config.js` 설정 파일에 명시한다

```js
import type { Config } from 'tailwindcss'

const config: Config = {
  content: [
    './src/pages/**/*.{js,ts,jsx,tsx,mdx}',
    './src/components/**/*.{js,ts,jsx,tsx,mdx}',
    './src/app/**/*.{js,ts,jsx,tsx,mdx}',
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
export default config
```

(4) Tailwind CSS를 사용하기 위한 CSS 파일 을 생성하고 tailwindcss를 import 한다

```css
/* globals.css */
@tailwind base;
@tailwind components;
@tailwind utilities;
```

(5) 프로젝트를 시작하고, tailwind 클래스를 적용한다

## 4. 자주 사용하는 Tailwind CSS 클래스

Tailwind CSS 클래스에는 색상, 폰트, 여백, 길이, 그리드, 플렉스, 테두리 등 여러가지 속성 정의 가능

- `bg-{color}`: 배경색을 지정하는 클래스로, 색상 코드 및 이름 정의 (e.g. bg-indigo-500)
- `text-{color}`: 텍스트 색상을 지정하는 클래스로, 색상 코드 및 이름 정의 (e.g. text-red-700)
- `font-{weight}`: 폰트 굵기를 지정하는 클래스로, 폰트 굵기 및 이름 정의 (e.g. font-bold, font-[700])
- `p-{size}, m-{size}`: 패딩 및 마진을 지정하는 클래스로, 각각의 패딩/마진 크기 정의 (e.g. p-2, m-2)
- `w-{size}, h-{size}`: 너비 및 높이를 지정하는 클래스로, 각각의 너비/높이 크기 정의 (e.g. w-1/2, h-64)
- `grid, flex`: 그리드와 플렉스를 정의할 때 사용하는 클래스 (e.g. grid grid-rows-4)
- `rounded-{radius}`: 요소를 둥글게 만드는 클래스 (e.g. rounded-lg)
- `border-{width}`: 테두리의 굵기를 정의할 때 사용하는 클래스 (e.g. border-4)

---

### 4.1 Utility-First Fundamentals

- TailwindCSS는 클래스 기반의 스타일링을 사용하며, 다양한 유틸리티 클래스를 통해 스타일 적용
- `bg-blue-500`: 배경색을 파란색으로 지정
- `text-white`: 텍스트 색상을 흰색으로 지정
- `p-4`: 패딩을 크기 4로 지정

---

### 4.2 Handling Hover, Focus, and Other States

- Tailwind CSS는 hover, focus 및 다른 상태에 대한 클래스들을 제공하여 상호작용하는 스타일 처리 가능
- `hover:bg-red-700` : 마우스 호버 상태에서 배경색 변경
- `focus:bg-red-700` : 포커스시 배경색 변경
- `focus:outline-none` : 포커스시 기본 아웃라인 제거

---

### 4.3 Responsive Design

- Tailwind는 반응형 디자인을 위한 클래스를 제공하며, 화면 크기에 따라 스타일 조절 가능
- e.g. `md:bg-red-500`은 중간 크기 화면에서 배경색을 빨강으로 변경

---

### 4.4 Dark Mode

- Tailwind 다크 모드를 구현하기 위한 클래스를 제공하며,
  - 다크 테마와 라이트 테마 간 전환 쉽게 처리할 수 있음
- `dark:bg-gray-800` : 다크모드에서 배경색을 어둡게 변경
- `dark:text-white` : 다크모드에서 텍스트 색상 변경

---

### 4.5 Reusing Styles

- Tailwind는 클래스를 조합해서 스타일을 재사용할 수 있으며, 사용자 정의 클래스를 생성할 수도 있음
- 자주 사용하는 클래스 코드들을 매번 컴포넌트마다 똑같이 복사하는 것이 아니라,
  - 하나의 클래스로 정리해서 재사용할 수 있음
- `btn-primary` : 미리 정의된 클래스를 조합하여 재사용가능한 스타일을 생성함

---

### 4.6 Adding Custom Styles

- `tailwind.config.js` 파일을 통해서 사용자 정의 스타일 및 커스텀 클래스를 추가할 수 있음
- colors에 원하는 색상을 추가하여, tailwind에서 사전에 정의한 색상이 아닌, 사용자가 원하는 색상으로 커스텀 가능
- e.g. `bg-blue`: 기존 tailwind 색상이 아닌, #1fb6ff 색상으로 나옴

---

### 4.7 Functions & Directives

- Tailwind의 함수와 디렉티브는 Tailwind를 더욱 확장 가능하게 만들어주고, 커스터마이징할 때 유용
- Tailwind에서는 테마 값을 동적으로 읽을 수 있는 theme() 함수와
  - 미디어 쿼리를 동적으로 생성할 수 있는 screen() 함수를 제공
- 또한, Tailwind만의 @규칙(at-rule) 디렉티브를 사용해서 조건에 따라 요소의 스타일을 조절 가능
  - `@tailwind`: HTML 요소에 Tailwind CSS 클래스를 동적으로 추가 가능
  - `@layer`: CSS 규칙을 레이어별로 구성할 수 있게 함
  - `@apply`: CSS 클래스를 동적으로 적용할 수 있음
  - `@config`: Tailwind CSS 구성 옵션 변경 가능
